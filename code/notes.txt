APP:
	<STARTING_SCREEN>:
		- if there is an existing game: 
			load data
			goto <ROUND_RESULT_SCREEN> 
		- else:
			button New Game -> initialize data; goto <IMAGE_METHOD_SCREEN>
			button Anylize Individual Image -> goto <IMAGE_METHOD_SCREEN>

	<IMAGE_METHOD_SCREEN>:
		- button Take Photo -> open camera, capture image, goto <BOARD_DETECTION>
		- button Select from Gallery -> open gallery, select image, goto <BOARD_DETECTION>

	<BOARD_DETECTION>:
		- process image (detect and rectify board)
		- goto <DETECTED_BOARD_SCREEN>

	<DETECTED_BOARD_SCREEN>:
		- display rectified board image
		- button Analyze Board -> run ML pipeline on cropped cells, goto <BOARD_MODEL_SCREEN>
		- button Retake/Reselect -> goto <IMAGE_METHOD_SCREEN>

	<BOARD_MODEL_SCREEN>:
		- display detected cell types, castle colors/ranks, card numbers/specials in a grid layout
		- allow user to tap on any cell to edit prediction
		- button Confirm -> goto <CALCULATE_SCORES>

	<CALCULATE_SCORES>:
		- calculate scores based on confirmed board state
		- goto <ROUND_RESULT_SCREEN>
	
	<ROUND_RESULT_SCREEN>:
		- display round number if in game mode
		- display scores
		- internaly save game state
		- button New Game -> initialize data; goto <IMAGE_METHOD_SCREEN>
		- button Next -> if round < 3: goto <IMAGE_METHOD_SCREEN>; else: popup with the winner message; goto <STARTING_SCREEN>

APP PIPELINE 
	1. locate and rectify the board 
	2. crop out every position on the board 
	3. for every position follow this diagram: 
		castle/card? 
			-(castle) -> classify castle (color and type of castle) 
			-(card) -> 
				has a number on it? 
					-(yes) -> crop out the number and classify the card based on it 
					-(no) -> classify the card based on the


NUMBER RECOGNITION CHALLENGE:
	Step 1: “Does the card have a number?”
	You can do this in multiple ways:
		Tiny CNN classifier
		OR classical OCR presence check
		OR edge/contrast heuristic (numbers are high-contrast)


IMPROVEMENT IDEAS:
	One architectural improvement (optional but powerful)
		Add confidence-based fallback logic:
			If a classifier's confidence < threshold:
				re-run alternative classifier or mark cell as “unknown” and request user confirmation
			

DATASET SIZE ESTIMATION:
	How much data you actually need (realistic numbers)
		Castle vs Card vs Empty
			~30-50 real photos total
		Heavy augmentation → thousands of samples
		
		Castle classification
			~5-10 photos per castle type per color
		Augment x50-100
		
		Digit classifier
			~10-20 real digit crops total
		Augment heavily
		
		Special cards
			~5 photos per card
		Augment x100
		
		Total real photos needed:
		➡️ ~100-150 photos, not thousands
